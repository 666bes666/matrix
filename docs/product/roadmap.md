# Продуктовый роадмап — Matrix

**Проект:** Matrix — веб-приложение матрицы компетенций
**Заказчик:** Управление сопровождения портала динамической инфраструктуры Сбер
**Команда:** Solo developer + AI
**Дедлайн:** Без жёсткого дедлайна
**Общая оценка:** ~26 недель (~6.5 месяцев)

## Phase 0: Фундамент — ~2 недели

### Ключевые результаты

- Инициализация репозитория (monorepo: backend + frontend)
- CI/CD-пайплайн (lint, тесты, сборка Docker-образов)
- Проектирование схемы БД (PostgreSQL, миграции через Alembic)
- Скелет проекта: FastAPI (backend) + React (frontend)
- Docker Compose для локальной разработки (backend, frontend, postgres, redis, celery-worker, celery-beat)
- Аутентификация: локальная БД пользователей, JWT-токены (access + refresh)
- Staging-окружение на Yandex Cloud
- Автоматические бэкапы PostgreSQL

### Критерии завершения

- `docker compose up` поднимает рабочее окружение за один шаг
- CI проходит: линтер, тесты, сборка образов — без ошибок
- Эндпоинт `/api/auth/login` выдаёт валидный JWT
- Миграции Alembic применяются к чистой БД без ошибок
- README содержит инструкции по запуску
- Staging-окружение доступно и деплоится автоматически при мерже в main
- Бэкап PostgreSQL выполняется ежедневно

### Результаты

**Дата завершения:** 2026-03-25
**Release:** v0.1.0 ([#5](https://github.com/666bes666/matrix/pull/5))
**Покрытие тестами:** 81.65% (14 тестов)
**Ключевые решения:**
- `pg_enum()` helper для корректного маппинга PostgreSQL enum values
- `bcrypt` напрямую вместо passlib (несовместимость с bcrypt 5.x)
- `NullPool` в тестах для изоляции event loop asyncpg
- Per-test Redis fixture для изоляции состояния кэша

### Зависимости

- Нет внешних зависимостей (стартовая фаза)

## Phase 1: Ядро MVP — ~6 недель

### Ключевые результаты

- Организационная структура: управления, отделы, команды (CRUD)
- Профили сотрудников: ФИО, должность, команда, контакты (CRUD)
- Каталог компетенций: категории, компетенции, уровни владения (0–4)
- Базовая оценка компетенций: TL оценивает сотрудников своей команды
- Табличное представление матрицы компетенций (сотрудники × компетенции)
- Ролевая модель доступа: Admin, Head, Department Head, TL, HR, Employee (6 ролей)

### Детализация по неделям

| Неделя | Задачи | Статус |
|--------|--------|--------|
| 3 | API организационной структуры, модели БД для org-дерева | **Завершено** ✓ |
| 4 | API профилей сотрудников, CRUD на фронте для сотрудников | В очереди |
| 5 | Каталог компетенций: модели, API, интерфейс управления | В очереди |
| 6 | Механизм оценки: TL → сотрудник, сохранение в БД | В очереди |
| 7 | Матрица компетенций: таблица на фронте, фильтрация | В очереди |
| 8 | RBAC: разграничение доступа по ролям, тесты авторизации | В очереди |

### Результаты недели 3

**Дата завершения:** 2026-03-25
**Release:** ([#6](https://github.com/666bes666/matrix/pull/6))
**Покрытие тестами:** 87.79% (32 теста: +18 новых)
**Реализовано:**
- Department API: 5 эндпоинтов (list, get, create, update, delete)
- Team API: 5 эндпоинтов, вложенные под /departments/{id}/teams
- Защита: 5 preset-отделов нельзя удалить (403), команды с сотрудниками — нельзя удалить (409)

### Критерии завершения

- Admin может создать структуру: управление → отдел → команда → сотрудник
- TL видит только свою команду, Head — всё управление
- Каталог содержит минимум 3 категории и 10 компетенций
- Матрица отображает оценки сотрудников по компетенциям в таблице
- Покрытие тестами: backend API ≥ 80%

### Зависимости

- Phase 0 (аутентификация, инфраструктура, БД)

## Phase 2: Оценка MVP — ~4 недели

### Ключевые результаты

- Workflow оценки 360°: самооценка + коллеги + TL + руководитель отдела (Department Head)
- Кампании оценки: создание, расписание, отслеживание статуса
- Целевые профили компетенций по ролям (target profiles)
- Gap-анализ: текущий уровень vs целевой по каждой компетенции
- Radar-чарт (лепестковая диаграмма) по сотруднику
- История оценок с возможностью просмотра динамики

### Детализация по неделям

| Неделя | Задачи |
|--------|--------|
| 9 | Модель кампании оценки, API создания/управления кампанией |
| 10 | Workflow 360°: назначение оценщиков, сбор оценок |
| 11 | Целевые профили, gap-анализ, API расчёта разрывов |
| 12 | Radar-чарт (Chart.js/Recharts), история оценок, UI кампаний |

### Критерии завершения

- Department Head может создать кампанию оценки, назначить участников, отследить прогресс
- Сотрудник проходит самооценку, получает агрегированный результат
- Gap-анализ показывает конкретные компетенции, требующие развития
- Radar-чарт корректно визуализирует профиль сотрудника
- История оценок доступна с фильтрацией по периоду

### Зависимости

- Phase 1 (каталог компетенций, профили сотрудников, RBAC)

## Phase 3: Аналитика MVP — ~4 недели

### Ключевые результаты

- Heatmap-визуализация: отдел × компетенция (средний уровень)
- Экспорт в Excel/PDF (матрица, отчёты по сотрудникам)
- Индивидуальные планы развития (IDP): создание, привязка к gap-анализу
- Учебные ресурсы: привязка к компетенциям (ссылки, курсы, книги)
- CSV/Excel импорт: массовая загрузка сотрудников, компетенций, оценок
- Telegram-уведомления: старт кампании, напоминания, завершение

### Детализация по неделям

| Неделя | Задачи |
|--------|--------|
| 13 | Heatmap (D3.js/Recharts), агрегация данных по отделам |
| 14 | Экспорт Excel (openpyxl) и PDF (WeasyPrint/ReportLab) |
| 15 | IDP: модель, CRUD, связь с gap-анализом; учебные ресурсы |
| 16 | CSV/Excel импорт; Telegram-бот для уведомлений |

### Критерии завершения

- Heatmap корректно отображает средние значения по отделам
- Excel-отчёт содержит матрицу с форматированием и стилями
- IDP содержит конкретные шаги с привязкой к компетенциям и дедлайнами
- Импорт 100+ сотрудников из Excel проходит за < 10 секунд
- Telegram-бот отправляет уведомления при старте кампании

### Зависимости

- Phase 2 (оценки, gap-анализ, кампании)

## Phase 4: Карьерные треки — ~3 недели

### Ключевые результаты

- Определение карьерных путей между ролями и отделами
- Требуемые компетенции для перехода на целевую позицию
- Визуализация карьерного пути (граф / дерево)
- Gap-анализ для целевой роли в другом отделе
- Дашборд готовности к переходу (transition readiness)

### Детализация по неделям

| Неделя | Задачи |
|--------|--------|
| 17 | Модель карьерных путей, API, связь ролей с компетенциями |
| 18 | Визуализация графа карьерных путей, gap-анализ для целевой роли |
| 19 | Дашборд готовности, рекомендации по развитию для перехода |

### Критерии завершения

- Определены минимум 3 карьерных трека с требованиями по компетенциям
- Сотрудник видит, каких компетенций ему не хватает для целевой позиции
- Граф карьерных путей интерактивен и отображает зависимости
- Дашборд показывает процент готовности к переходу

### Зависимости

- Phase 1 (организационная структура, каталог компетенций)
- Phase 2 (оценки, целевые профили, gap-анализ)

## Phase 5: Финализация — ~3 недели

### Ключевые результаты

- Доработка UI/UX: единообразие, отзывчивость, доступность
- Оптимизация производительности: кэширование, пагинация, индексы БД
- Аудит безопасности: XSS, CSRF, SQL-injection, проверка JWT
- Пользовательская документация (руководство пользователя)
- Деплой: staging → production (Docker, Nginx, PostgreSQL)
- Обучение пользователей (демо-сессии)

### Детализация по неделям

| Неделя | Задачи |
|--------|--------|
| 20 | UI/UX полировка, оптимизация запросов, lazy loading |
| 21 | Аудит безопасности, нагрузочное тестирование, документация |
| 22 | Staging-деплой, smoke-тесты, обучение пользователей, production |

### Критерии завершения

- Lighthouse score: Performance ≥ 80, Accessibility ≥ 90
- Среднее время ответа API < 200ms для основных эндпоинтов
- Аудит безопасности пройден без критических замечаний
- Пользовательская документация покрывает все основные сценарии
- Production-окружение стабильно работает под нагрузкой

### Зависимости

- Все предыдущие фазы (0–4)
