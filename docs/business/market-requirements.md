# Market Requirements Document (MRD) — Matrix

**Продукт:** Matrix — веб-приложение матрицы компетенций для IT-подразделений
**Версия документа:** 1.0
**Дата:** 2026-03-24
**Статус:** Draft

## 1. Обзор рынка

### 1.1 Глобальный рынок Talent Management

Глобальный рынок систем управления талантами (Talent Management Software) оценивается в $10–12 млрд (2025) с прогнозируемым CAGR 12–14% до 2030 года. Сегмент Competency Management Systems (CMS) — подмножество этого рынка — растёт опережающими темпами за счёт перехода организаций к skills-based моделям.

Российский рынок HR-tech оценивается в 30–40 млрд руб., при этом сегмент управления компетенциями остаётся фрагментированным: большинство компаний используют Excel или внутренние разработки без единого стандарта.

### 1.2 Ключевые тренды

| Тренд | Описание | Влияние на Matrix |
|-------|----------|-------------------|
| **Skills-based organizations** | Переход от управления по должностям к управлению по навыкам. Компании строят skills taxonomies и skills graphs | Прямое — Matrix решает именно эту задачу для IT-команд |
| **Internal mobility** | Фокус на удержание через внутренние карьерные перемещения вместо найма. Снижение стоимости замены сотрудника | Matrix визуализирует карьерные треки и gap-анализ |
| **Data-driven HR** | HR-решения на основе данных, а не интуиции. Аналитика навыков, прогнозирование оттока | Matrix накапливает историю оценок — основа для аналитики |
| **Импортозамещение (RU)** | Уход западных вендоров (SAP, Workday, Oracle) с российского рынка. Потребность в локальных решениях | Стратегическое окно возможностей для Matrix |
| **Self-hosted / on-premise** | Крупные банки и госкорпорации требуют размещения на собственной инфраструктуре (ФЗ-152, стандарты ЦБ) | Matrix изначально проектируется с поддержкой self-hosted |

| **1С:Управление по целям** | Российский HRM-модуль | $$ (лицензия + внедрение) | Российский вендор, интеграция с 1С | Устаревший UX, привязка к 1С-экосистеме, нет фокуса на IT | Не специализирован на IT-компетенциях; тяжёлый legacy-стек |
| **Excel / Google Sheets** | Самописное | Бесплатно | Привычно, гибко, нулевой порог входа | Нет версионирования оценок, нет визуализации, ошибки при масштабировании, нет ролевой модели | Текущий подход в 80%+ компаний — именно он создаёт потребность в Matrix |

### 2.2 Выводы из конкурентного анализа

1. **Западные корпоративные решения** (SAP, Workday, Oracle) ушли из российского рынка — образовалась ниша.
2. **Специализированные SaaS** (Skills Base, AG5) не имеют русской локализации и не предоставляют self-hosted вариант.
3. **Российские HR-системы** (1С, Websoft HCM) не специализируются на IT-компетенциях и предлагают устаревший UX.
4. **Excel** — де-факто стандарт — не масштабируется, не обеспечивает аналитику и историю.

**Вывод:** Рынок открыт для специализированного российского решения по управлению IT-компетенциями.

## 5. Позиционирование Matrix

### 5.1 Что Matrix НЕ пытается быть

- **Не полноценная HRM-система** — не заменяет 1С:ЗУП, SAP HR или Workday для кадрового учёта, расчёта зарплат, отпусков.
- **Не LMS (Learning Management System)** — не хранит курсы, не управляет обучением (но может интегрироваться).
- **Не Performance Review платформа** — не заменяет OKR/KPI-системы (Lattice, 15Five), хотя данные оценки компетенций могут использоваться в ревью.
- **Не универсальное HR-решение** — не пытаемся покрыть все роли (продажи, маркетинг, производство).

### 5.2 Что Matrix делает лучше всех

**Одна вещь, сделанная правильно: матрица компетенций + карьерные треки для IT-команд.**

### 5.3 Ключевые преимущества

| Преимущество | Описание | В сравнении с конкурентами |
|-------------|----------|---------------------------|
| **Простота** | Запуск за минуты, не за месяцы. Интуитивный UI без обучения | SAP/Workday — внедрение 6–18 мес.; Excel — прост, но не масштабируется |
| **Специализация на IT** | Предустановленные шаблоны для IT-ролей (Backend, Frontend, DevOps, SRE, QA, Data, ML). Понимание IT-специфики грейдов | Корпоративные HRM — generic; специализированные SaaS — ориентированы на manufacturing |
| **Локализация** | Русский язык, российские реалии (грейды, структуры подразделений), соответствие ФЗ-152 | Западные SaaS — английский, GDPR, нет понимания российского рынка |
| **Self-hosted опция** | Развёртывание на инфраструктуре клиента (Docker / Kubernetes). Данные не покидают контур | Skills Base, AG5, Competency Core — только SaaS |
| **Открытая таксономия** | Полностью настраиваемые навыки, категории, шкалы, роли. Никаких жёстких фреймворков | Competency Core — фиксированная библиотека; AG5 — ограниченная кастомизация |
| **API-first** | REST API для интеграции с любыми внутренними системами | Excel — нет API; корпоративные HRM — сложные API с дорогими коннекторами |

### 5.4 Формула позиционирования

> **Для** IT-руководителей и HR-партнёров в технологических компаниях,
> **которые** хотят систематизировать управление навыками команды,
> **Matrix** — это специализированный инструмент матрицы компетенций,
> **который** визуализирует навыки, отслеживает рост и строит карьерные треки.
> **В отличие от** корпоративных HRM (избыточных и дорогих) и Excel (немасштабируемого),
> **Matrix** прост во внедрении, специализирован на IT и может работать на вашей инфраструктуре.

## 6. Барьеры входа и риски

### 6.1 Конкуренция

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Российские HRM-вендоры (Websoft, 1С) добавят модуль компетенций | Высокая | Среднее | Двигаться быстрее; специализация > универсальность. К моменту их выхода Matrix уже будет иметь базу пользователей и экспертизу в IT-компетенциях |
| Появление аналогичного стартапа | Средняя | Среднее | First-mover advantage внутри Сбера; быстрое расширение на банковский сектор |
| Крупный tech (Яндекс, VK) создаст внутренний инструмент и опенсорсит | Низкая | Высокое | Фокус на enterprise-фичах (аудит, SSO, SLA), которые опенсорс-проекты обычно не покрывают |

### 6.2 Adoption (принятие пользователями)

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Сотрудники не хотят проходить оценку (воспринимают как контроль) | Высокая | Высокое | UX, ориентированный на пользу сотруднику (карьерный трек, рекомендации), а не только на контроль руководителя. Прозрачность: сотрудник видит свой профиль |
| Руководители не заполняют матрицу регулярно | Высокая | Высокое | Напоминания, интеграция в существующие процессы (ревью), минимизация времени на заполнение |
| Миграция с Excel — болезненна | Средняя | Среднее | Импорт из Excel в один клик; шаблоны для типовых структур |

### 6.3 Комплайнс и безопасность

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Требования ФЗ-152 (персональные данные) | Гарантировано | Высокое | Self-hosted вариант; данные в контуре клиента; шифрование at rest и in transit |
| Требования ЦБ для банковского сектора (ГОСТ, аттестация) | Высокая (для банков) | Высокое | Аудит-логирование всех действий; ролевая модель доступа; подготовка к аттестации |
| Утечка данных о навыках сотрудников | Низкая | Высокое | RBAC, аудит, шифрование, пентесты |

### 6.4 Технические риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Масштабирование при росте числа пользователей | Средняя | Среднее | Архитектура с расчётом на горизонтальное масштабирование с первого дня |
| Зависимость от конкретного стека | Низкая | Низкое | Стандартные технологии (Python, PostgreSQL, Docker) без vendor lock-in |
| Сложность интеграции с корпоративными системами | Средняя | Среднее | API-first подход; документированный REST API; стандартные протоколы (SAML, LDAP) |

## Приложение A: Оценка объёма рынка (Россия)

| Сегмент | Количество компаний | Средний чек (год) | TAM |

| Крупные банки и финтех | ~50 | 3–10 млн руб. | 150–500 млн руб. |
| Tech-компании (DevOps/SRE) | ~200 | 0.5–2 млн руб. | 100–400 млн руб. |

**SOM (Serviceable Obtainable Market):** ~10% от SAM = 12–42 млн руб. в первый год коммерциализации.

## Приложение B: Ключевые метрики успеха продукта

| Метрика | Описание | Целевое значение (год 1) |
|---------|----------|--------------------------|
| **Активные организации** | Количество компаний, использующих Matrix | 10+ (включая Сбер) |
| **MAU (Monthly Active Users)** | Ежемесячно активные пользователи | 1 000+ |
| **Retention (организации)** | % организаций, продливших подписку | >80% |
| **Time to Value** | Время от регистрации до первой заполненной матрицы | <1 день |
| **NPS** | Net Promoter Score среди IT-руководителей | >40 |
